# programming-notes

A repository of technical notes in Markdown format focused on web development, Python, automation, and environment setup tools. The goal of this space is to keep quick references, examples, and practical reminders for different kinds of projects.

## Repository Structure

At the moment, the repository is organized as topic-based files in the root directory rather than application folders. Each file works as an independent note.

```text
programming-notes/
|-- Conceptos React .md
|-- Create SSH.md
|-- Docker y Airflow.md
|-- Flask Python.md
|-- pyautogui.md
|-- README.md
|-- Start Proyect React.md
`-- virtual environment Python.md
```

## File Overview

### React

- [Conceptos React .md](./Conceptos%20React%20.md): introduction to React, its advantages, functional and class components, JSX, props, state, hooks, and events.
- [Start Proyect React.md](./Start%20Proyect%20React.md): basic commands to create and run a React application with `create-react-app`.

### Python

- [Flask Python.md](./Flask%20Python.md): short explanation of Flask, installation steps, and a minimal web app example.
- [pyautogui.md](./pyautogui.md): notes about automation with PyAutoGUI, coordinate capture, and locating elements through image recognition.
- [virtual environment Python.md](./virtual%20environment%20Python.md): guide to creating, activating, deactivating, and managing Python virtual environments.

### DevOps and Tools

- [Docker y Airflow.md](./Docker%20y%20Airflow.md): initial steps for working with Docker Desktop and Apache Airflow, including a basic folder structure and useful commands.
- [Create SSH.md](./Create%20SSH.md): placeholder file for future notes about SSH key setup. It currently has no content.

## Organization Logic

The current structure is grouped by topic:

- Frontend: React concepts and project startup notes.
- Backend and automation: Flask, PyAutoGUI, and Python virtual environment management.
- Infrastructure: Docker and Airflow.
- Environment setup: future notes such as SSH.

## Recommended Use

- Use this repository as a quick reference for commands, core concepts, and starter project structures.
- If the repository grows, a natural next step would be to move the files into folders such as `react/`, `python/`, `devops/`, and `tools/`.
- It may also be helpful to standardize the language used across the notes for better consistency.
